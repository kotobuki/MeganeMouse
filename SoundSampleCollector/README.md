# MeganeMouse Sound Sample Collector

MeganeMouseに音声クリック機能を追加するためのサンプル収集ツールです。

AtomS3Rに接続したマイク（PDMマイクまたはAtomic Echo Base）で連続した音声を録音し、USB Serial経由でPCにWAVファイル（16kHz / 16bit / モノラル）として保存します。収集したWAVファイルはEdge Impulseにアップロードし、ツール上で個別のサンプルに分割してモデルの学習に使用できます。

## ハードウェア要件

- **M5Stack AtomS3R**
- **USB-Cケーブル**（データ通信対応のもの）
- **マイク**（以下のいずれかを使用）：
  - **PDMマイク**（例：Unit Mini PDM等をPort Aに接続）
  - **M5Stack Atomic Echo Base**（AtomS3Rの底面に装着）

## セットアップ

### Arduino側

1. Arduino IDEで以下のライブラリをインストール：
   - `M5Unified`（依存ライブラリ含む）
2. ボード設定：
   - ボード：`M5AtomS3R`（M5Stackボードパッケージ）
   - PSRAM：`OPI PSRAM`
   - USB CDC On Boot：`Enabled`
3. 使用するマイクに合わせて`SoundSampleCollector.ino`の先頭行を変更：
   - PDMマイクを使用する場合：`#define USE_PDM_MIC 1`
   - Atomic Echo Baseを使用する場合：`#define USE_PDM_MIC 0`
4. スケッチをコンパイル・書き込み

> **注意：** 本スケッチでは`M5EchoBase`ライブラリは**使用しません**。これは、同ライブラリが使用するレガシーI2Sドライバが、M5Unifiedの新しいI2Sドライバと競合するためです。本スケッチでは`M5.Mic`APIを使用し、ES8311コーデックを利用する場合はI2C経由で直接初期化を行います。

### PC側

```bash
pip install pyserial
```

## 使い方

### 1. PCで受信スクリプトを起動

```bash
python receive_samples.py --port /dev/cu.usbmodem*  # macOS
python receive_samples.py --port /dev/ttyACM0       # Linux
python receive_samples.py --port COM3               # Windows
```

### 2. 連続録音

本スケッチは**連続録音モード**を採用しています。1回のクリックで60秒間（デフォルト）の連続録音を行い、終了後に自動的にPCへ転送します。

**操作方法：**

| 操作               | 待機中           | 録音中         |
| ------------------ | ---------------- | -------------- |
| **クリック**       | 録音を開始       | 録音を途中停止 |
| **ダブルクリック** | クラス切替       | -              |
| **長押し**         | マイクゲイン切替 | -              |

**録音の流れ：**

```
クリック→カウントダウン※→60秒間連続録音→自動転送→完了
                                ↑
                    途中でクリックすれば途中停止可能
                    （録音済みの部分だけが転送される）
```

> **※カウントダウンの動作について：**
>
> - **PDMマイク使用時**：スピーカーが存在しないため、LCD画面上の数字と点滅（視覚表示）のみでカウントダウンを行います。
> - **Atomic Echo Base使用時**：内蔵スピーカーを使用し、画面表示に加えてビープ音（3・2・1）が鳴ります。

### 3. Edge Impulseでの分割

1. 録音されたWAVファイルをEdge Impulseにアップロード
2. Edge Impulseのデータ分割機能（Split sample）を使用して個別のサンプルに分割
3. 不適切な区間があれば手動で削除

この方式により、ユーザーはカウントダウンに合わせて正確なタイミングで発音し続ける必要がなくなります。自然なペースで音を出し、後からEdge Impulse上で整理できます。

### 4. LCD表示

- **紺**：待機中（クラス名・録音数・ゲイン・録音時間を表示）
- **紺＋大きい数字**：カウントダウン中（3→2→1）
- **赤**：録音中（経過秒数とプログレスバーを表示）
- **オレンジ**：送信中（プログレスバー表示）
- **緑**：完了

### 5. 出力フォルダ構成

```
samples/
  click_left/
    0001.wav    # 60秒の連続録音
    0002.wav
    ...
  click_right/
    0001.wav
    ...
  noise/
    0001.wav
    ...
```

各WAVファイルは16kHz/16bit/モノラルです。

## 録音のコツ

### ワークフロー例

1. ダブルクリックで`click_left`クラスを選択
2. クリックで録音開始
3. 60秒間、自分のペースで左クリック用の音を繰り返し発音
4. 自動転送を待つ
5. Edge Impulseにアップロード→分割→不要な区間を削除
6. `click_right`、`noise` も同様に繰り返す

### noiseクラスの録音

noiseクラスは特に簡単です。録音を開始したら、何もせずに60秒間放置するだけで環境音が自動的に収集されます。複数の環境（静かな部屋、テレビの前、外出先など）で録音するとモデルのロバスト性が向上します。

### 録音時間の変更

デフォルトの60秒では長すぎる場合や、より長い連続録音が必要な場合は、ソースコード中の `CONTINUOUS_DURATION_SEC` を変更してください。

```cpp
#define CONTINUOUS_DURATION_SEC  60  // 変更：30, 120 など
```

AtomS3RのPSRAM（8MB）に収まる範囲であれば、最大約200秒まで設定可能です。

## 技術的な補足

### アーキテクチャ

**PDM マイクの場合（`USE_PDM_MIC 1`）**

```
M5Unified ─── Display（M5.Display）
          ├── Button （M5.BtnA）
          └── Mic    （M5.Mic）     ← 録音（I2S：PDMインターフェース）
                 │
            PDM マイク（Port A 等に接続）
```

**Atomic Echo Base（ES8311）の場合(`USE_PDM_MIC 0`）**

```
M5Unified ─── Display（M5.Display）
          ├── Button （M5.BtnA）
          ├── Speaker（M5.Speaker） ← ビープ音（I2S 共有、Mic と排他）
          └── Mic    （M5.Mic）     ← 録音（I2S 共有、Speaker と排他）
                 │
            I2S（GPIO 5,6,7,8）← Speaker / Mic で切替
                 │
            ES8311 コーデック ← I2C 直接制御（GPIO 38,39, addr 0x18）
              ├── DAC → NS4150B アンプ → スピーカー
              └── ADC ← MEMS マイク
```

### 連続録音の仕組み

60秒の録音は内部的に1秒ごとのチャンクに分割して実行されます。M5.Micのリングバッファ（128ms）がチャンク間のデータを保持するため、音声データに欠落は生じません。チャンク間のLCD更新時間（~10ms）分のオーディオはリングバッファに蓄積され、次のチャンクの先頭で読み出されます。

### メモリ使用量

PSRAMから録音用バッファを確保します。PSRAMは他のライブラリー等も使用するため、表に掲載しているPSRAM残量が全て使用できるわけではありません。

| 録音時間 | 録音用バッファサイズ | PSRAM残量 |
| -------- | -------------------- | --------- |
| 30秒     | 960 KB               | 約7 MB    |
| 60秒     | 1.88 MB              | 約6 MB    |
| 120秒    | 3.75 MB              | 約4 MB    |
| 200秒    | 6.25 MB              | 約1.7 MB  |

### ES8311 クロック設定（Atomic Echo Base使用時）

Atomic Echo BaseにはMCLKピンがないため、ES8311はBCLKから内部クロックを生成します（係数テーブル`{1024000, 16000}`を使用）。

## トラブルシューティング

### 録音データが無音

- **PDMマイク使用時**：マイクのピン接続（DAT/CLK）が正しいか確認してください。
- **Atomic Echo Base使用時**：ES8311の初期化ログで`ES8311 found at 0x18`を確認してください。
- マイクゲインを上げてみてください（待機中にボタン長押しで切替）。

### 転送に時間がかかる

USB CDCではbaud rate設定は仮想的なものです。60秒分（約1.92MB）の転送には通常数十秒かかります。

### メモリ不足

`CONTINUOUS_DURATION_SEC`を小さくしてください。AtomS3RのPSRAMは8MBですが、M5Unifiedやその他のライブラリもメモリを使用します。
